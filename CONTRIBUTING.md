# コントリビューション

コントリビューターおよび AI エージェントは、以下のコミット・タグ規約に従ってください。

## コミットメッセージ（Conventional Commits）

このリポジトリでは [Conventional Commits](https://www.conventionalcommits.org/) に従ってコミットメッセージを書いてください。

### 形式

```
<type>[optional scope]: <description>

[optional body]

[optional footer(s)]
```

### 主な type

| type       | 説明 |
|-----------|------|
| `feat`    | 新機能 |
| `fix`     | バグ修正 |
| `docs`    | ドキュメントのみの変更 |
| `style`   | コードの意味に影響しない変更（フォーマット等） |
| `refactor`| リファクタリング |
| `test`    | テストの追加・修正 |
| `chore`   | ビルド・CI・ツール等の変更 |

### 例

```
feat(about): トップページに自己紹介セクションを追加
fix: モバイルでナビが折り返されない問題を修正
docs: CONTRIBUTING にコミット規約を追加
```

## プルリクエスト（PR）

- **タイトル**: **英語**で書いてください。
- **本文（説明）**: **原則日本語**で書いてください。

## コミット前レビュー

コミット作成前に、以下の観点で**レビュー担当（サブエージェント）**を実行してください。

| 観点 | 確認内容 |
|------|----------|
| **自然言語の文法** | 追加・変更した文章（README、コメント、UI 文言、ドキュメント等）の表記・文法の確認 |
| **セキュリティ** | 変更が機密情報の露出、インジェクション、不適切な依存の追加等を招いていないかの確認 |
| **CI** | 該当するワークフロー（例: `.github/workflows/`）が変更内容で通るか、必要なチェックが揃っているかの確認 |

レビューで問題があれば修正してからコミットしてください。プロジェクトに `pre-commit-review` スキルがある場合は、その手順に従って実行します。

## タグ・バージョン（Calendar Versioning）

リリースタグには [Calendar Versioning（CalVer）](https://calver.org/) を採用します。

### 形式

- **YYYY.MM** — 年と月（例: `2025.3`）
- **YYYY.MM.MICRO** — 同月内の複数リリース用（例: `2025.3.1`, `2025.3.2`）

月はゼロパディングなし（`1`〜`12`）で表記します。

### 例

```
2025.3      # 2025年3月のリリース
2025.3.1    # 同月の 1 回目
2025.12.2   # 2025年12月の 2 回目
```

### タグの付け方

```bash
git tag 2025.3
# または
git tag 2025.3.1
git push origin 2025.3.1
```
