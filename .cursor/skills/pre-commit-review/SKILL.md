# コミット前レビュー（レビュー担当サブエージェント）

コミット作成前に実行するレビューです。次の 3 観点を順に確認し、問題があれば指摘・修正してからコミットしてください。

## 1. 自然言語の文法

- **対象**: 追加・変更した文章（README、コメント、UI 文言、ドキュメント、PR 本文・コミットメッセージ案など）
- **確認**: 表記ゆれ、誤字脱字、文法、敬体・常体の統一、句読点
- **出力**: 問題箇所と修正案を簡潔に示す。問題がなければ「問題なし」と記載

## 2. セキュリティ観点

- **確認項目**:
  - 機密情報（API キー、パスワード、トークン）のハードコードやログ出力の有無
  - ユーザー入力をそのまま出力・実行していないか（XSS、インジェクション等）
  - 新規依存の追加がある場合、既知の脆弱性や信頼性
  - 環境変数・シークレットの扱いが適切か
- **出力**: 懸念があれば指摘と推奨対応。なければ「問題なし」

## 3. CI の確認

- **対象**: `.github/workflows/` 内のワークフロー
- **確認**: 変更したファイル・言語・ディレクトリに応じて、どのワークフローが走るか把握し、以下を確認する
  - ワークフローが存在し、トリガー（paths / branches 等）が妥当か
  - 必要なチェック（ビルド、テスト、lint 等）が含まれているか
  - 変更内容が CI を壊していないか（型エラー、テスト失敗の可能性）
- **実行**: 可能であれば該当する CI 相当のコマンドをローカルで実行して確認（例: `bun run build`, `bunx astro check`）
- **出力**: 実行結果または「手元で CI 相当を実行して確認推奨」など

## 実行の流れ

1. 上記 1 → 2 → 3 の順で確認する
2. 各観点の結果を短くまとめる
3. 指摘がある場合は修正を提案または実施し、修正後に再度必要な部分だけ確認する
4. 問題が解消したらコミットしてよい旨を伝える
